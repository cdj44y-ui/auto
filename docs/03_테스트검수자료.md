### 2.3 버그 상세

#### BUG-001: 관리자 대시보드 로딩 오류

**증상:** 관리자 대시보드 접속 시 빈 화면 표시

**원인:** 컴포넌트 import 경로 오류

**해결 방법:** import 경로 수정

**영향 범위:** 관리자 대시보드 페이지

---

#### BUG-002: useAuth import 경로 오류

**증상:** 로그인 후 인증 상태 유지 실패

**원인:** useAuth 훅의 import 경로가 잘못됨

**해결 방법:** 
```typescript
// 수정 전
import { useAuth } from "@/hooks/useAuth";

// 수정 후
import { useAuth } from "@/_core/hooks/useAuth";
```

**영향 범위:** 모든 인증 관련 기능

---

#### BUG-003: 직원 추가 Dialog 미표시

**증상:** '직원 추가' 버튼 클릭 시 아무 반응 없음

**원인:** Dialog 상태 관리 로직 누락

**해결 방법:** useState를 사용한 Dialog 열림/닫힘 상태 관리 추가

**영향 범위:** 직원 관리 페이지

## 3. 사용자 매뉴얼 초안

### 3.1 시스템 접속

#### 3.1.1 로그인 방법

1. 웹 브라우저에서 시스템 URL에 접속합니다.
2. 로그인 페이지가 표시됩니다.
3. 이메일과 비밀번호를 입력합니다.
4. '로그인' 버튼을 클릭합니다.
5. 역할에 따라 해당 대시보드로 이동합니다.

> **테스트 환경:** 테스트 환경에서는 역할 버튼(직원, 관리자, 자문사, 개발자)을 클릭하여 자동으로 테스트 계정 정보가 입력됩니다.

#### 3.1.2 역할별 접근 권한

| 역할 | 접근 가능 메뉴 |
|-----|--------------|
| 관리자 (Admin) | 모든 메뉴 |
| HR 담당자 | 직원 관리, 근태 관리, 휴가 승인 |
| 재무 담당자 | 급여 관리, 급여 통계 |
| 일반 직원 | 개인 대시보드, 휴가 신청, 급여 명세서 조회 |

### 3.2 직원 관리

#### 3.2.1 직원 목록 조회

1. 좌측 메뉴에서 '직원 관리'를 클릭합니다.
2. 전체 직원 목록이 표시됩니다.
3. 검색창에서 이름 또는 이메일로 검색할 수 있습니다.
4. 부서 필터를 사용하여 특정 부서만 조회할 수 있습니다.

#### 3.2.2 직원 추가

1. '직원 추가' 버튼을 클릭합니다.
2. 신규 직원 등록 Dialog가 열립니다.
3. 필수 항목을 입력합니다:
   - 이름 (필수)
   - 이메일 (필수)
   - 부서 (필수)
   - 직급 (필수)
4. 선택 항목을 입력합니다:
   - 전화번호
   - 입사일
5. '등록' 버튼을 클릭합니다.

#### 3.2.3 직원 수정

1. 수정할 직원의 행에서 연필 아이콘(✏️)을 클릭합니다.
2. 직원 정보 수정 Dialog가 열립니다.
3. 필요한 정보를 수정합니다.
4. '저장' 버튼을 클릭합니다.

#### 3.2.4 직원 삭제

1. 삭제할 직원의 행에서 휴지통 아이콘(🗑️)을 클릭합니다.
2. 확인 Dialog가 표시됩니다.
3. '삭제' 버튼을 클릭하여 삭제를 확정합니다.

> **주의:** 삭제된 직원 정보는 복구할 수 없습니다.

#### 3.2.5 엑셀 일괄 업로드

1. '엑셀 업로드' 버튼을 클릭합니다.
2. '템플릿' 버튼을 클릭하여 양식을 다운로드합니다.
3. 엑셀 파일에 직원 정보를 입력합니다.
4. 파일을 업로드합니다.
5. 미리보기에서 데이터를 확인합니다.
6. '일괄 등록' 버튼을 클릭합니다.

### 3.3 급여 관리

#### 3.3.1 급여 대장 조회

1. 좌측 메뉴에서 '급여 관리'를 클릭합니다.
2. 기간을 선택합니다 (년/월).
3. 해당 기간의 급여 대장이 표시됩니다.

#### 3.3.2 급여 명세서 발송

1. 급여 대장에서 발송할 직원을 선택합니다.
2. '명세서 발송' 버튼을 클릭합니다.
3. 발송 확인 Dialog에서 '발송' 버튼을 클릭합니다.
4. 발송 결과가 표시됩니다.

### 3.4 근태 관리

#### 3.4.1 출퇴근 기록

1. 대시보드에서 '출근하기' 버튼을 클릭합니다.
2. 현재 시간이 출근 시간으로 기록됩니다.
3. 퇴근 시 '퇴근하기' 버튼을 클릭합니다.

#### 3.4.2 근태 현황 조회

1. 좌측 메뉴에서 '근태 관리'를 클릭합니다.
2. 전체 직원의 근태 현황이 표시됩니다.
3. 날짜 범위를 선택하여 특정 기간을 조회할 수 있습니다.

describe("auth.logout", () => {
  it("clears the session cookie and reports success", async () => {
    const { ctx, clearedCookies } = createAuthContext();
    const caller = appRouter.createCaller(ctx);

    const result = await caller.auth.logout();

    expect(result).toEqual({ success: true });
    expect(clearedCookies).toHaveLength(1);

      sameSite: "none",
      httpOnly: true,
      path: "/",
    });
  });
});

### 6.2 결론

근태관리시스템(AES)은 모든 핵심 기능에 대한 테스트를 통과했습니다. RBAC 기반 권한 관리가 정상적으로 작동하며, UI 컴포넌트들도 예상대로 동작합니다. 실제 이메일 발송 기능은 현재 시뮬레이션으로 구현되어 있으며, 프로덕션 환경에서는 SMTP 또는 SendGrid 연동이 필요합니다.
